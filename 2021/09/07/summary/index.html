Sep-01-2021 HSP Java讲座笔记
< Head First Java>

简介

1. 程序就是文件，是让计算机进行使用的。

2. 用windows自带程序运行java

a. 先用记事本生成 name.java 文件
b. 在文件所在地址栏直接输入cmd + 回车，直接启动此处的终端
c. 在cmd中输入javac name.java.，进行编译，生成name.class的文件。
class文件才是系统能读懂的。
d. 在cmd中输入java name，来运行程序。
e. 上下光标可以调用cmd中的前面出现的指令。
f. tab键可以自动补全。

3. Platform-independent 跨平台性

source.java (source code) -编译- 通过javac compiler生成 .class文件(不同于source code里的class) – 转成计算机能读的code ‘bytecode’ -运行- 虚拟机virtual machine来运行bytecode
The bytecode is platform-independent

4. 结构：source code - class file - methods - statement

其中method中的main 是一切代码开始的地方。一个project中可以有多个class，但是只有一个main method。

5. method中可以做什么：

a. statements: declarations, assignments, method calls, etc
b. loop: for and while and do-while
c. branching: if/else test

6. OOP - objective-oriented programming 面向对象的程序

class file is a blueprint for an object. Everything in Java is an object.

9. 开发细节说明

  • 一个源文件中最多只能有一个public的类，其他类的个数不限。
  • .java文件编译后，每个类会生成对应的.class文件。即使这些class都写在一个.java文档里，编译后也会生成不同的.class文件。
  • 写着很多class的.java的文件需要以public类的名字命名，每一个生成的.class文件名会自动以每个class的名命名。
  • 可以将main方法写在非public类中，然后指定运行非public类，这样入口方法就是非public的main方法。

1. 数据类型 (特殊的：long, float, char)

Java数据类型分为基本数据类型和引用数据类型。其中基本数据类型包括8种：

  • 数值型：整数类型- byte（1个字符），short(2个字符)，int（4），long(8); 浮点（小数）类型：float（4），double（8）

  • 字符型：char（2），存放单个字符；

  • boolean型：boolean（1），存放true，false
    引用数据类型包括：类class，接口interface，数组[]
    string属于class

  • Java整型常量默认为int，声明long型常量须后加l或者L，除非不足以表示大数，才用long
    long n1 = 1L; 因为是将右侧值赋给左侧，如果不加L或者F，就自动默认为int8位或者double8位，如果左侧的类型比它范围大，可以；
    范围小就装不下。如 int n1 = 1L (no)； long n1 = 1（yes）；

  • bit是计算机中的最小存储单位，byte是基本存储单位，1 byte = 8 bit。

  • 浮点数 = 符号位 + 指数位 + 尾数位
    尾数部分可能丢失，造成精度损失，所以小数都是近似值
    所以当对运算结果是小数的进行相等判断时，要小心，因为小数的结果都是近似值，应该以两个数的插值的绝对值在某个精度范围内判断

  • Java浮点型常量默认位double，声明float型常量须后加f或者F
    float n1 = 1.1F;

  • 字符类型

    1. char，可以直接存放数字，不用引号，但是输出时会输出数字对应的字符（unicode编码）
      char c1 = 97; 这条语句打印出来的结果是a
      char c2 = ‘a’； 如果打印 (int) c2，结果就是97
    2. （重点）char用单引号，string用双引号 char c2 = ‘a’
    3. 转义字符\不占位置，char c3 = ‘\t’是可以的，但不是输出t，而是对应的含义，此处为制表位
    4. char类型是可以进行运算的，相当于一个整数
      System.out.println(‘a’ + 10); 结果是107

  • boolean的值只能是true或false

2. 数据类型自动转换

  1. 数据类型按精度（容量）从小到大排序为：（小的类型可以放到大的里面去）
    char - int - long - float - double 从左到右放 eg: int a = ‘c’; 对的
    byte - short - int - long - float - double eg: double b = 80; 对的
  2. 有多种类型的数据混合运算时，系统首先将所有数据转换成容量最大的那种数据类型，然后进行运算
    eg: int n1 = 10;
    float n2 = n1 + 1.1 //错误
    float n2 = n1 + 1.1F //正确
    double n2 = n1 + 1.1 // 正确
  3. 当把精度（容量）大的数据类型赋值给精度小的数据类型时，会报错，反之可以进行自动类型转换。
    eg: int n2 = 1.1 //错误
  4. （byte，short）和 char之间不会相互自动转换
    eg： byte b1 = 10; // 正确，没有默认为10是int，因为如果具体数值是在-128至127之间，如果是就可以用byte
    char c1 = b1; //错误
  5. byte，short和char三者单独或者混合可以进行计算，只是首先变为int类型
    eg: byte b1 = 1;
    byte b2 = 2;
    short s1 = 2;
    short s2 = b1 + s1; //错误，因为b1 + s1 自动转换为int类型
    byte b3 = b1 + b2; // 错误，虽然为同一类型，但是也要转为int
    eg: short s = 12; //ok
    s = s - 9; //错误，因为已经转换为了int
  6. boolean不参与自动转换，排序中无它

3. 数据类型强制转换

  1. 强制转换是自动转换的逆过程，将容量大的数据类型转换为容量小的数据类型。要使用强制符()，但可能造成精度降低或者溢出
    eg1: int n1 = (int)1.9; //double -> int 结果是1，造成精度损失
    eg2：int n2 = 1000;
    byte n3 = (byte) n2; // int -> byte  结果是溢出，无关联
  2. 强制符只针对最近的操作数有效，往往会使用小括号优先升级
    eg1：int n1 = (int)103.5+21.9; 这里只会将10的类型转换为int，结果仍然是double
    所以要使用 int n1 = (int)(103.5+21.9)
  3. char类型可以保存int的常量值（输出的就是根据unicode得到的字母），但是不能保存int的变量值
    eg1： char c1 = 100; // ok
    eg2: int n1 = 100;
      char c2 = n1' // wrong
  char c3 = (char)n1; //ok

7. 进制

  1. 二进制：0,1，满2进1，以0b或0B开头
  2. 十进制：0-9，满10进1
  3. 八进制：0-7，满8进1，以0开头
  4. 十六进制：0-9及A（10）- F（15），满16进1，以0X或0x开头，此处字母不区分大小写

  • 二进制转十进制
    规则：从最低为（右边）开始，将每位上的数提取出来，乘以2的（位数-1）次方，然后求和
    0b1011 = 1x2的0次方 + 1x2的1次方 + 0 + 1x2的3次方 = 11

  • 八进制转十进制
    规则： 从最低为（右边）开始，将每位上的数提取出来，乘以8的（位数-1）次方，然后求和

  • 十六进制转十进制
    规则： 从最低为（右边）开始，将每位上的数提取出来，乘以16的（位数-1）次方，然后求和

  • 十进制转二进制
    规则： 将该数不断除以2，直到商为0，然后将每步的余数倒过来

    结果不够8位的，前方要填充0；且加进制标识

  • 十进制转八进制
    规则： 将该数不断除以8，直到商为0，然后将每步的余数倒过来

  • 十进制转十六进制
    规则： 将该数不断除以16，直到商为0，然后将每步的余数倒过来

  • 二进制转八进制
    规则：从低位开始，将二进制数每三位一组，转成对应的八进制数
    eg：0b11（3）010（2）101（5） –> 0325

  • 二进制转十六进制
    规则：从低位开始，将二进制数每四位一组，转成对应的十六进制数

  • 八进制转二进制
    规则：将八进制的每一位，转成对应的一个3位的二进制数

  • 十六进制转二进制
    规则：将十六进制的每一位，转成对应的一个4位的二进制数

8. 位运算

  • 原码、反码、补码的规则 （重点）

    1. 二进制的最高位时符号位：0代表正数，1代表负数 （旋转90°）
    2. 正数的原码、反码和补码都一样（三码合一）
    3. 负数的反码 = 它的原码符号位不变，其他位取反 （0变1，1变0）
    4. 负数的补码 = 它的反码+1， 负数的反码 = 它的补码-1
    5. 0的反码、补码都是0
    6. java中的数都是有符号的
    7. 在计算机运算的时候，都是以补码的方法来运算的
    8. 当我们看运算结果的时候，要看它的原码

  • 7个位运算符： &按位与, |按位或, ^按位异或, ~按位取反, >>, <<, >>>

  • 按位与 & ：两组二进制，对应的位数上全为1的，结果为1，否则为0

  • 按位或 | ：两组二进制，对应的位数上一个为1的，结果为1，否则为0

  • 按位异或 ^ ：两组二进制，对应的位数上一个为0，一个为1的，结果为1，否则为0

  • 按位取反 ~： 0变1，1变0.
